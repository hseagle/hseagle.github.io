2021-07-13

安装pip_search来解决pip search无法使用的痛点

  1. 安装patroni
  2. 安装consul
  3. 停止通过systemctl启动的postgresql和pgbouncer

patroni

patronictl -c /tmp/pg-1.yml list my-ha-cluster

手工切换

patronictl -c /tmp/pg-1.yml failover my-ha-cluster

查看当前数据库是master还是standby, 如果返回值为true，则是standby, 反则反之

select pg_is_in_recovery();

pgbouncer

select * from pg_shadow;
select * from pg_authid;

2021-07-15

在生产环境尝试安装consul

  1. 安装consul
  2. patroni安装postgresql
  3. 配置pgbouncer
  4. 同步程序链接指向pgbouncer
sudo yum -y install python36-pip python36-virutalenv
 sudo yum localinstall -y ./postgresql13-libs-13.2-1PGDG.rhel7.x86_64.rpm ./postgresql13-13.2-1PGDG.rhel7.x86_64.rpm ./postgresql13-server-13.2-1PGDG.rhel7.x86_64.rpm ./postgresql13-contrib-13.2-1PGDG.rhel7.x86_64.rpm
 mkdir /opt/app/patroni-env
 chown -R postgres:postgres /opt/app/patroni-env
 cd /opt/app/patroni-env
sudo -u postgres python3 -m venv .
source bin/activate

添加用户

pip install --upgrade pip --index-url http://mirrors.ops.ctripcorp.com/pypi-latest/web/simple/ --trusted-host mirrors.ops.ctripcorp.com

pip install psycopg2-binary --index-url http://mirrors.ops.ctripcorp.com/pypi-latest/web/simple/ --trusted-host mirrors.ops.ctripcorp.com

pip install patroni[consul] --index-url http://mirrors.ops.ctripcorp.com/pypi-latest/web/simple/ --trusted-host mirrors.ops.ctripcorp.com
 adduser consul --system --no-create-home

修改配置文件

sudo -u postgres /opt/app/patroni-env/bin/patronictl -c /opt/app/patroni-env/pg-1.yml edit-config
sudo -u postgres /opt/app/patroni-env/bin/patronictl -c /opt/app/patroni-env/pg-1.yml list
